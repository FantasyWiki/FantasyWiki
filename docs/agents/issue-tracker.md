# Issue tracker

This repository uses **GitHub Issues** as the source of truth.

- Repository: `FantasyWiki/FantasyWiki`
- Tracker type: GitHub
- Primary CLI: `gh`

## How skills should operate

- Read issues from GitHub Issues in this repository.
- Create/update/triage issues via GitHub labels and comments.
- Use normal GitHub issue workflow; do not write issue files under `.scratch/`.

## Typical commands

- List open issues: `gh issue list --state open`
- View one issue: `gh issue view <number>`
- Create issue: `gh issue create --title "<title>" --body-file <file>`
